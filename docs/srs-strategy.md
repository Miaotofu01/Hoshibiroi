# 星拾 记忆策略文档

## 概述

星拾采用 **FSRS-5 DSR 记忆模型**（Difficulty / Stability / Retrievability），调度策略参考 Anki 源码设计，参数针对**语言学习场景**（日均 5-20 新词）调优。

核心原则：
- 每次评分**立即修改记忆状态**——不攒到"毕业"才提交
- 新词学习阶段**不惩罚**长期记忆参数——第一次见不认识是正常的
- 复习词每次犹豫**都留下痕迹**——Hard→Hard→Good 和直接 Good 结果不同

---

## 记忆三参数

| 参数 | 范围 | 含义 |
|------|------|------|
| **D** (Difficulty) | 1-10 | 词的内在难度。越高越难，增长越慢 |
| **S** (Stability) | >0 | 记忆稳定性。R 从 100% 降到 90% 所需天数 |
| **R** (Retrievability) | 0-1 | 当前可回忆概率。取决于上次复习后过了多久 |

三者关系：
```
R(t, S) = (1 + 0.2346 × t/S)^(−0.5)

当 t = S 时，R = 0.9（稳定性的定义）
当 t = 0 时，R = 1.0（刚复习完）
当 t ≫ S 时，R → 0（彻底遗忘）
```

## 四级评分

| 按键 | Grade | 含义 | 典型场景 |
|------|-------|------|---------|
| 不认识 | 1 (Again) | 完全没想起来 | 看到词，脑中一片空白 |
| 模糊 | 2 (Hard) | 想起来了但费劲 | 犹豫了几秒才记起意思 |
| 认识 | 3 (Good) | 正常回忆 | 看到词，一秒内反应出来 |
| 简单 | 4 (Easy) | 毫不费力 | 已经非常熟悉的词 |

## 各评分对记忆状态的影响

以下以**非首次、非同日**的正常复习为例，假设 D≈5，S≈12，上次复习 5 天前（R≈0.92）：

### 不认识 (Again)

```
D：5.0 → 6.6  (+1.6)  大幅上升 — 没想起来说明比预想的难
S：12  → 2.1  (−83%)  坍塌 — 记忆残余只有原来的 1/5
reviewCount：→ 0      归零 — 从头再爬
下次间隔：~2 天
```

走 `nextForgetStability` 分支。稳定性不会降到 0——保留了部分残余记忆，比完全重新学略好。

### 模糊 (Hard)

```
D：5.0 → 5.8  (+0.8)  小幅上升 — 挣扎说明难度比预期高
S：12  → 17   (+42%)  增长但被惩罚 — hardPenalty=0.40 把 SInc 压到正常的 40%
reviewCount：→ +1
下次间隔：~13 天
```

### 认识 (Good)

```
D：5.0 → 5.0  (不变)  难度不变
S：12  → 23   (+92%)  正常翻倍 — 无奖惩
reviewCount：→ +1
下次间隔：~18 天
```

### 简单 (Easy)

```
D：5.0 → 4.2  (−0.8)  难度下降 — 比预期的容易
S：12  → 46   (+283%) 大幅增长 — easyBonus=2.99 给了三倍加速
reviewCount：→ +1
下次间隔：~35 天
```

## 特殊场景

### 首次复习

`reviewCount=0` 时，用固定初始稳定性，不触发增长公式：

| 首次评分 | 初始 S | 首间隔 |
|---------|--------|--------|
| 不认识 | 0.4 | ~1 天 |
| 模糊 | 1.2 | ~1 天 |
| 认识 | 3.2 | ~2 天 |
| 简单 | 15.7 | ~12 天 |

### 同日复习

同一会话内的多次评分，用短时记忆公式而非增长公式：

```
S' = S × e^(0.517 × (grade − 3 + 0.662))
```

| 同日评分 | S 倍率 | 效果 |
|---------|--------|------|
| 不认识 | ×0.50 | 跌一半 |
| 模糊 | ×0.84 | 微跌 |
| 认识 | ×1.41 | 涨四成 |
| 简单 | ×2.36 | 翻倍 |

### Lapse 后恢复

点了"不认识"后 reviewCount 归零，但系统记录了 `lastReviewedAt > 0`。之后再点"认识"时，不会触发 `initStability`（首次初始化），而是基于 lapse 后的残余稳定性继续——惩罚被保留，不会因为随后点"认识"就洗白。

## 复习流程

### 新词（学习阶段）

```
step 1（首次见面）
  ┌ 不认识 → 回到 step 1 [不提交]
  ├ 模糊   → 重复 step 1 [不提交]
  ├ 认识   → 进入 step 2 [不提交] — 10分钟后再次确认
  └ 简单   → 直接毕业 ✅ [提交 grade=4]
  
step 2（确认）
  ┌ 不认识 → 回到 step 1 [不提交]
  ├ 模糊   → 重复 step 2 [不提交]
  ├ 认识   → 毕业 ✅ [提交 grade=3]
  └ 简单   → 毕业 ✅ [提交 grade=4]
```

学习阶段的评分**不修改长期记忆状态**（匹配 Anki 的设计：第一次见不认识是正常的）。毕业时才首次调用 `schedule()`。

### 复习词

```
每次弹出
  ┌ 不认识 → 立即提交 grade=1 [D涨, S塌] → 重新插入队列
  ├ 模糊   → 立即提交 grade=2 [D涨, S微涨] → 重新插入队列（最多3次后强制毕业）
  ├ 认识   → 提交 grade=3 [正常增长] → 毕业
  └ 简单   → 提交 grade=4 [加速增长] → 毕业
```

复习词的**每一次评分都立即调 `schedule()`**。连续几次 Hard 会在状态机上自然累积惩罚——不需要手工"综合评分"。
